# নেটওয়ার্ক টপোলজি (Network Topology)

## 📖 টপোলজি কী?

**টপোলজি (Topology)** হলো নেটওয়ার্কে ডিভাইসগুলো কিভাবে সাজানো এবং একে অপরের সাথে যুক্ত আছে তার বিন্যাস বা কাঠামো।

### দুই ধরনের টপোলজি:
1. **Physical Topology**: তারগুলো আসলে কিভাবে সাজানো
2. **Logical Topology**: ডেটা কিভাবে প্রবাহিত হয়

---

## 🔵 ১. বাস টপোলজি (Bus Topology)

### গঠন
সব ডিভাইস একটি মূল ক্যাবলের সাথে যুক্ত।

### ডায়াগ্রাম
```
     Terminator                                    Terminator
         |                                              |
         ════════════════════════════════════════════════
              |         |         |         |
            [PC1]     [PC2]     [PC3]     [PC4]
```

### কাজের পদ্ধতি
- একটি backbone cable থাকে
- সব ডিভাইস এই cable এ connected
- ডেটা উভয় দিকে যায়
- প্রাপক ডিভাইস ডেটা গ্রহণ করে

### সুবিধা ✅
| সুবিধা | বিবরণ |
|--------|--------|
| সহজ | ইনস্টল করা সহজ |
| সস্তা | কম ক্যাবল লাগে |
| ছোট নেটওয়ার্ক | ছোট নেটওয়ার্কের জন্য ভালো |

### অসুবিধা ❌
| অসুবিধা | বিবরণ |
|----------|--------|
| Single Point of Failure | মূল ক্যাবল নষ্ট হলে পুরো নেটওয়ার্ক বন্ধ |
| Collision | একসাথে ডেটা পাঠালে সংঘর্ষ হয় |
| স্লো | বেশি ডিভাইস হলে ধীর হয়ে যায় |
| ট্রাবলশুট কঠিন | সমস্যা খুঁজে বের করা কঠিন |

## ⭐ ২. স্টার টপোলজি (Star Topology)

### গঠন
সব ডিভাইস কেন্দ্রীয় ডিভাইসের (Hub/Switch) সাথে যুক্ত।

### ডায়াগ্রাম
```
          [PC1]
            |
    [PC4]---[Switch]---[PC2]
            |
          [PC3]
```

### কাজের পদ্ধতি
- কেন্দ্রে Hub বা Switch থাকে
- প্রতিটি ডিভাইসের আলাদা কানেকশন
- সব ডেটা কেন্দ্র দিয়ে যায়

### সুবিধা ✅
| সুবিধা | বিবরণ |
|--------|--------|
| নির্ভরযোগ্য | একটি ডিভাইস নষ্ট হলে বাকিগুলো চলে |
| সহজ ট্রাবলশুট | সমস্যা সহজে খুঁজে পাওয়া যায় |
| সহজ এক্সপ্যানশন | নতুন ডিভাইস যোগ করা সহজ |
| ভালো পারফরম্যান্স | কম collision |

### অসুবিধা ❌
| অসুবিধা | বিবরণ |
|----------|--------|
| কেন্দ্রীয় নির্ভরতা | Hub/Switch নষ্ট হলে সব বন্ধ |
| বেশি ক্যাবল | প্রতিটি ডিভাইসের জন্য আলাদা ক্যাবল |
| খরচ বেশি | Hub/Switch কিনতে হয় |

### ⭐ সবচেয়ে জনপ্রিয় টপোলজি!
আজকের বেশিরভাগ LAN এই টপোলজি ব্যবহার করে।

---

## 🔄 ৩. রিং টপোলজি (Ring Topology)

### গঠন
প্রতিটি ডিভাইস দুটি প্রতিবেশী ডিভাইসের সাথে যুক্ত, বৃত্তাকার গঠন।

### ডায়াগ্রাম
```
        [PC1]
       /     \
    [PC4]   [PC2]
       \     /
        [PC3]
```

### কাজের পদ্ধতি
- ডেটা একদিকে ঘুরে ঘুরে যায়
- Token Passing পদ্ধতি ব্যবহার করে
- যার কাছে Token আছে সে ডেটা পাঠাতে পারে

### সুবিধা ✅
| সুবিধা | বিবরণ |
|--------|--------|
| সমান সুযোগ | সবাই সমান সুযোগ পায় |
| No Collision | Token থাকায় সংঘর্ষ নেই |
| ভালো পারফরম্যান্স | ভারী লোডেও ভালো কাজ করে |

### অসুবিধা ❌
| অসুবিধা | বিবরণ |
|----------|--------|
| Single Point of Failure | একটি ডিভাইস/ক্যাবল নষ্ট হলে সব বন্ধ |
| ধীর | ডেটা সব ডিভাইস ঘুরে যায় |
| যোগ করা কঠিন | নতুন ডিভাইস যোগ করতে নেটওয়ার্ক বন্ধ করতে হয় |

### Dual Ring
সমস্যা সমাধানে দুটি রিং ব্যবহার করা হয় - একটি নষ্ট হলে অন্যটি কাজ করে।

## 🌐 ৪. মেশ টপোলজি (Mesh Topology)

### গঠন
প্রতিটি ডিভাইস অন্য সব ডিভাইসের সাথে সরাসরি যুক্ত।

### ডায়াগ্রাম
```
      [PC1]───────[PC2]
        |  \     /  |
        |    \ /    |
        |    / \    |
        |  /     \  |
      [PC4]───────[PC3]
```

### প্রকারভেদ

#### Full Mesh
- প্রতিটি ডিভাইস অন্য সবার সাথে যুক্ত
- সংযোগ সংখ্যা = n(n-1)/2
- ৪টি ডিভাইস = ৬টি কানেকশন

#### Partial Mesh
- কিছু ডিভাইস সব সাথে যুক্ত
- কিছু ডিভাইস কিছু সাথে যুক্ত
- খরচ কম, তবে ফুল রিডান্ডেন্সি নেই

### সুবিধা ✅
| সুবিধা | বিবরণ |
|--------|--------|
| অত্যন্ত নির্ভরযোগ্য | একটি পথ বন্ধ হলে অন্য পথ আছে |
| ফাস্ট | সরাসরি কমিউনিকেশন |
| সিকিউর | ডেটা সরাসরি যায় |
| No Traffic Problem | ট্রাফিক বিভিন্ন পথে ভাগ হয় |

### অসুবিধা ❌
| অসুবিধা | বিবরণ |
|----------|--------|
| অনেক ক্যাবল | প্রচুর ক্যাবল লাগে |
| খুব ব্যয়বহুল | সবচেয়ে বেশি খরচ |
| জটিল | সেটআপ ও মেইনটেন্যান্স কঠিন |
| স্কেলেবল নয় | বেশি ডিভাইস যোগ করা কঠিন |

### ব্যবহার
- WAN (Internet backbone)
- মিলিটারি নেটওয়ার্ক
- ব্যাংকিং সিস্টেম
- যেখানে উচ্চ নির্ভরযোগ্যতা দরকার

## 🌳 ৫. ট্রি টপোলজি (Tree/Hierarchical Topology)

### গঠন
Star এবং Bus টপোলজির সংমিশ্রণ। গাছের মতো শাখা-প্রশাখা।

### ডায়াগ্রাম
```
              [Main Switch]
             /      |      \
        [Switch1] [Switch2] [Switch3]
         /  \      /  \       /  \
      [PC1][PC2][PC3][PC4] [PC5][PC6]
```

### সুবিধা ✅
| সুবিধা | বিবরণ |
|--------|--------|
| স্কেলেবল | সহজে বড় করা যায় |
| গ্রুপিং | বিভাগ অনুযায়ী গ্রুপ করা যায় |
| ম্যানেজমেন্ট | কেন্দ্রীয় ম্যানেজমেন্ট সহজ |

### অসুবিধা ❌
| অসুবিধা | বিবরণ |
|----------|--------|
| রুট নির্ভরতা | রুট নোড নষ্ট হলে সব বন্ধ |
| ক্যাবলিং জটিল | অনেক ক্যাবল লাগে |
| Maintenance | রক্ষণাবেক্ষণ কঠিন |

---

## 🔀 ৬. হাইব্রিড টপোলজি (Hybrid Topology)

### সংজ্ঞা
দুই বা ততোধিক টপোলজির সংমিশ্রণ।

### উদাহরণ
```
    [Star Network 1]──────[Backbone Bus]──────[Star Network 2]
          |                    |                    |
      [Switch1]            [Router]            [Switch2]
       /  |  \                                  /  |  \
    [PC1][PC2][PC3]                         [PC4][PC5][PC6]
```

### ব্যবহার
- বড় অর্গানাইজেশন
- বিভিন্ন বিভাগে বিভিন্ন টপোলজি
- WAN এবং LAN সংযোগ
