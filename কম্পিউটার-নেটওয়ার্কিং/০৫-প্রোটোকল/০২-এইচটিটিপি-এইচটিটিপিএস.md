# HTTP এবং HTTPS প্রোটোকল

## 📖 HTTP কী?

**HTTP (HyperText Transfer Protocol)** হলো World Wide Web এর foundation protocol যা web browser এবং web server এর মধ্যে communication করে।

```
Web Browser                          Web Server
    │                                     │
    │ ────── HTTP Request ──────────────→ │
    │        GET /index.html              │
    │                                     │
    │ ←───── HTTP Response ────────────── │
    │        200 OK + HTML content        │
    │                                     │
```

## 🔧 HTTP Basics

### Request-Response Model

```
┌─────────────────────────────────────────────────────────┐
│                  HTTP Workflow                           │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  1. User types URL: www.example.com                     │
│                                                          │
│  2. Browser creates HTTP Request:                       │
│     GET / HTTP/1.1                                      │
│     Host: www.example.com                               │
│                                                          │
│  3. Server processes request                            │
│                                                          │
│  4. Server sends HTTP Response:                         │
│     HTTP/1.1 200 OK                                     │
│     Content-Type: text/html                             │
│     <html>...</html>                                    │
│                                                          │
│  5. Browser renders page                                │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### Port Numbers

```
HTTP:  Port 80   (unencrypted)
HTTPS: Port 443  (encrypted)

http://www.example.com → Port 80
https://www.example.com → Port 443
```

## 📤 HTTP Request Structure

```
┌─────────────────────────────────────────────────────────┐
│                  HTTP Request                            │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  Request Line:                                          │
│  GET /products/item.html HTTP/1.1                       │
│  │     │                   │                            │
│  │     │                   └── HTTP Version             │
│  │     └── Resource Path (URI)                          │
│  └── Method (GET, POST, etc.)                           │
│                                                          │
│  Headers:                                               │
│  Host: www.example.com                                  │
│  User-Agent: Mozilla/5.0 (Windows NT 10.0)             │
│  Accept: text/html,application/json                     │
│  Accept-Language: en-US,bn-BD                          │
│  Cookie: session=abc123                                 │
│                                                          │
│  Body (for POST/PUT):                                   │
│  {"name": "John", "email": "john@example.com"}         │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### HTTP Methods

| Method | Purpose | Body? | Example |
|--------|---------|-------|---------|
| GET | Data পড়া | না | Page load |
| POST | Data পাঠানো | হ্যাঁ | Form submit |
| PUT | Data update | হ্যাঁ | Profile update |
| DELETE | Data মুছা | না | Account delete |
| PATCH | Partial update | হ্যাঁ | Edit field |
| HEAD | Header only | না | Check if exists |
| OPTIONS | Allowed methods | না | CORS check |

```
GET vs POST:

GET /search?q=networking HTTP/1.1
    ↑ Data in URL (visible, limited size)

POST /login HTTP/1.1
Content-Type: application/x-www-form-urlencoded

username=john&password=secret123
    ↑ Data in body (hidden, larger size)
```

## 🔐 HTTPS কী?

**HTTPS (HTTP Secure)** হলো HTTP এর encrypted version যা **TLS/SSL** ব্যবহার করে।

```
HTTP (Unsafe):
Browser ───── Plain Text ─────→ Server
         Anyone can read!

HTTPS (Secure):
Browser ───── 🔒 Encrypted 🔒 ─────→ Server
         Only browser & server can read
```

```
┌─────────────────────────────────────────────────────────┐
│              HTTPS Benefits                              │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  🔒 Encryption                                          │
│     Data encrypted, hackers can't read                  │
│                                                          │

│     (No fake websites)                                  │
│                                                          │
│  📝 Integrity                                           │

│                                                          │
│  🔍 SEO Boost                                           │
```

## 🔑 TLS/SSL Handshake

HTTPS connection কীভাবে establish হয়:

```
Browser                              Server
   │                                    │
   │ ──── 1. Client Hello ────────────→ │
   │      "Hi! I support TLS 1.3,       │
   │       these cipher suites..."      │
   │                                    │
   │ ←─── 2. Server Hello ──────────── │
   │      "Hi! Let's use TLS 1.3,       │
   │       this cipher suite"           │
   │      + Server Certificate          │
   │                                    │
   │ ──── 3. Verify Certificate ─────   │
   │      (Check CA signature,          │
   │       domain match, expiry)        │
   │                                    │
   │ ──── 4. Key Exchange ───────────→ │
   │      (Generate shared secret)      │
   │                                    │
   │ ←─── 5. Finished ─────────────── │
   │                                    │
   │ ═══ Encrypted Communication ═════ │
   │                                    │
```

### Certificate Chain

```
Root CA (Trusted by browser)
    │
    └── Intermediate CA
            │
            └── Your Website Certificate
                    │
                    └── www.example.com
                        (Domain verified)

Browser trusts Root CA
→ Trusts Intermediate
→ Trusts Website Certificate
→ Trusts website!
```

## 📊 HTTP Versions

```
┌─────────────────────────────────────────────────────────┐
│              HTTP Version Evolution                      │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  HTTP/0.9 (1991)                                        │
│  └── Simple, GET only, no headers                       │
│                                                          │
│  HTTP/1.0 (1996)                                        │
│  └── Headers added, multiple methods                    │
│  └── One request per connection                         │
│                                                          │
│  HTTP/1.1 (1997) - Still widely used!                  │
│  └── Keep-Alive (reuse connection)                      │
│  └── Chunked transfer                                   │
│  └── Host header (virtual hosting)                      │
│                                                          │
│  HTTP/2 (2015)                                          │
│  └── Binary protocol (faster)                           │
│  └── Multiplexing (multiple requests, one connection)   │
│  └── Header compression                                 │
│  └── Server Push                                        │
│                                                          │
│  HTTP/3 (2022)                                          │
│  └── Based on QUIC (UDP-based)                         │
│  └── Faster connection setup                            │
│  └── Better for mobile/unstable networks               │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### HTTP/1.1 vs HTTP/2

```
HTTP/1.1:
┌──────────┐     ┌──────────┐     ┌──────────┐
│Request 1 │────→│Response 1│     │          │
└──────────┘     └──────────┘     │          │
┌──────────┐     ┌──────────┐     │ Server   │
│Request 2 │────→│Response 2│     │          │
└──────────┘     └──────────┘     │          │
┌──────────┐     ┌──────────┐     │          │
│Request 3 │────→│Response 3│     │          │
└──────────┘     └──────────┘     └──────────┘
    Sequential (one at a time)


HTTP/2:
┌──────────┐
│Request 1 │─────┐
│Request 2 │─────┼────→ Server ────→ All responses
│Request 3 │─────┘                   multiplexed!
└──────────┘
    Parallel (all at once)
```

## 🍪 Cookies and Sessions

### Cookies

```
HTTP is stateless (মনে রাখে না)
Cookies help remember users!

1. First visit:
Browser ───→ Server
Server ───→ Set-Cookie: session=abc123

2. Next visits:
Browser ───→ Cookie: session=abc123 ───→ Server
                                         "আহা! তোমাকে চিনি!"
```

### Cookie Types

```
┌─────────────────────────────────────────────────────────┐
│                  Cookie Types                            │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  Session Cookie:                                        │
│  - Browser বন্ধ করলে মুছে যায়                          │
│  - Login state                                          │
│                                                          │
│  Persistent Cookie:                                     │
│  - Expiry date থাকে                                     │
│  - "Remember me" feature                                │
│                                                          │
│  Third-party Cookie:                                    │
│  - অন্য domain এর (ads, tracking)                       │
│  - Privacy concern!                                     │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

## 🛠️ HTTP Headers

### Common Request Headers

| Header | Purpose | Example |
|--------|---------|---------|
| Host | Target server | www.example.com |
| User-Agent | Browser info | Mozilla/5.0... |
| Accept | Expected content type | text/html |
| Accept-Language | Preferred language | en-US,bn |
| Authorization | Login credentials | Bearer token123 |
| Cookie | Session data | session=abc |
| Content-Type | Body format | application/json |
| Cache-Control | Caching behavior | no-cache |

### Common Response Headers

| Header | Purpose | Example |
|--------|---------|---------|
| Content-Type | Body format | text/html |
| Content-Length | Body size | 1234 |
| Set-Cookie | Store cookie | session=xyz |
| Cache-Control | Cache rules | max-age=3600 |
| Location | Redirect URL | /new-page |
| Access-Control-Allow-Origin | CORS | * |

## 💻 Testing HTTP

### Using curl

```bash
# Simple GET
curl https://www.example.com

# GET with headers shown
curl -v https://www.example.com

# Just headers
curl -I https://www.example.com

# POST with data
curl -X POST -d "name=John" https://api.example.com/users

# POST JSON
curl -X POST \
  -H "Content-Type: application/json" \
  -d '{"name":"John"}' \
  https://api.example.com/users

# With authentication
curl -u username:password https://api.example.com
curl -H "Authorization: Bearer TOKEN" https://api.example.com
```

### Using Browser DevTools

```
Chrome/Firefox: F12 → Network tab

দেখতে পাবেন:
├── Request URL
├── Request Method
├── Status Code
├── Headers (Request & Response)
├── Response body
├── Timing (latency)
└── Cookies
```

```
        📝 Integrity
```
