# নেটওয়ার্ক ডিজাইন (Network Design)

## 📖 Network Design কী?

**Network Design** হলো organization এর requirements অনুযায়ী efficient, scalable, এবং secure network architecture তৈরির process।

```
Requirements → Design → Implementation → Maintenance
      │           │           │              │
      └───────────┴───────────┴──────────────┘
            Network Lifecycle
```

## 🏗️ Three-Tier Hierarchical Model

Enterprise network এর সবচেয়ে proven design model।

```
┌─────────────────────────────────────────────────────────┐
│                                                          │
│                     ┌───────────┐                       │
│                     │   Core    │                       │
│                     │  Layer    │     ← High-speed      │
│                     └─────┬─────┘       backbone        │
│                    ┌──────┴──────┐                      │
│                    │             │                       │
│              ┌─────┴─────┐ ┌─────┴─────┐               │
│              │Distribution│ │Distribution│  ← Policy,   │
│              │   Layer   │ │   Layer   │    Routing    │
│              └─────┬─────┘ └─────┬─────┘               │
│                    │             │                       │
│        ┌───────────┼───────────┐ │                      │
│        │           │           │ │                       │
│    ┌───┴───┐   ┌───┴───┐   ┌───┴─┴─┐                   │
│    │Access │   │Access │   │Access │    ← End user      │
│    │ Layer │   │ Layer │   │ Layer │      connectivity  │
│    └───┬───┘   └───┬───┘   └───┬───┘                   │
│        │           │           │                        │
│      Users       Users       Users                      │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### Layer Functions

| Layer | Function | Devices |
|-------|----------|---------|
| **Core** | High-speed transport | High-end routers, L3 switches |
| **Distribution** | Policy, routing, filtering | L3 switches, routers |
| **Access** | User connectivity | L2 switches, APs |

```

├─────────────────────────────────────────────────────────┤
│                                                          │
│           ┌──────────────────────────┐                  │
│           │   Core/Distribution      │                  │
│           │   (Collapsed)            │                  │

│           │            │            │                    │

│                                                          │

│                                                          │
└─────────────────────────────────────────────────────────┘
```

```
.11+ - DHCP range
```

## 🔄 Redundancy Design

### Network Redundancy

```
Redundancy at Each Layer:

                 ┌─────────┐
            ┌────│ Router1 │────┐
            │    └────┬────┘    │     WAN
            │         │         │
      ──────┼─────────┼─────────┼──────
            │         │         │
            │    ┌────┴────┐    │
            └────│ Router2 │────┘
                 └─────────┘
                      │
         ┌────────────┼────────────┐
         │            │            │
    ┌────┴────┐  ┌────┴────┐  ┌────┴────┐
    │  Core1  │══│ ══════ │══│  Core2  │  ← Dual Core
    └────┬────┘  │        │  └────┬────┘
         ├───────┤        ├───────┤
         │       │        │       │
    ┌────┴────┐  │   ┌────┴────┐  │
    │  Dist1  │──┘   │  Dist2  │──┘      ← Dual Distribution
    └────┬────┘      └────┬────┘
         │                │
         └───────┬────────┘
                 │
            ┌────┴────┐
            │ Access  │                   ← Dual uplinks
            └─────────┘
```

### Gateway Redundancy (HSRP/VRRP)

```
HSRP Example:

              Virtual IP: 10.0.1.1 (Gateway)
                        │
         ┌──────────────┴──────────────┐
         │                             │
    ┌────┴────┐                   ┌────┴────┐
    │ Router1 │                   │ Router2 │
    │ 10.0.1.2│                   │ 10.0.1.3│
    │ ACTIVE  │                   │ STANDBY │
    └────┬────┘                   └────┬────┘
         │                             │
    ─────┴─────────────────────────────┴─────
                      │
                  End Users
                  (Gateway: 10.0.1.1)

If Router1 fails → Router2 becomes ACTIVE
Users don't notice!
```

```

- কী ধরনের firewall?

Justify your choices.
```

## 🎓 মনে রাখুন

```
Design Principles:

1. Start simple, scale as needed
2. Always plan for redundancy
3. Segment with VLANs
4. Document everything
5. Think about security from day 1
6. Plan for 2-3 years growth

Three-Tier Model:
Core → Fast transport
Distribution → Policy & routing
Access → User connection
```
