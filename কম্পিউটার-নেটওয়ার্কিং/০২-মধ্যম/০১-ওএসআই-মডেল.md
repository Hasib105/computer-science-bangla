# OSI মডেল (OSI Model)

## 📖 OSI মডেল কী?

**OSI (Open Systems Interconnection) Model** হলো নেটওয়ার্ক কমিউনিকেশনকে বুঝতে এবং ডিবাগ করতে ব্যবহৃত একটি ৭-লেয়ারের কনসেপচুয়াল ফ্রেমওয়ার্ক।

### সহজ উদাহরণ:
```
আপনি যখন www.google.com টাইপ করেন:
- ব্রাউজার (Layer 7) রিকোয়েস্ট তৈরি করে
- ডেটা এনক্রিপ্ট হয় (Layer 6)
- সেশন তৈরি হয় (Layer 5)
- TCP কানেকশন হয় (Layer 4)
- IP অ্যাড্রেস যোগ হয় (Layer 3)
- MAC অ্যাড্রেস যোগ হয় (Layer 2)
- বিট আকারে তারে/Wi-Fi তে যায় (Layer 1)
```

---

## 🧱 OSI এর ৭টি লেয়ার

```
┌─────────────────────────────────────────────┐
│  Layer 7: Application    (অ্যাপ্লিকেশন)      │ ← ইউজার এখানে
├─────────────────────────────────────────────┤
│  Layer 6: Presentation   (প্রেজেন্টেশন)       │
├─────────────────────────────────────────────┤
│  Layer 5: Session        (সেশন)              │
├─────────────────────────────────────────────┤
│  Layer 4: Transport      (ট্রান্সপোর্ট)       │
├─────────────────────────────────────────────┤
│  Layer 3: Network        (নেটওয়ার্ক)         │
├─────────────────────────────────────────────┤
│  Layer 2: Data Link      (ডেটা লিংক)         │
├─────────────────────────────────────────────┤
│  Layer 1: Physical       (ফিজিক্যাল)         │ ← তার/সিগন্যাল
└─────────────────────────────────────────────┘
```

---

## 📊 প্রতিটি লেয়ারের বিস্তারিত

### 🔵 Layer 7: Application Layer (অ্যাপ্লিকেশন)

```
[ব্রাউজার] [ইমেইল ক্লায়েন্ট] [FTP সফটওয়্যার]
     ↓           ↓              ↓
         Application Layer
```

| বৈশিষ্ট্য | বিবরণ |
|----------|--------|
| কাজ | ইউজার ইন্টারফেস, অ্যাপ সার্ভিস |
| প্রোটোকল | HTTP, HTTPS, FTP, SMTP, DNS, SSH |
| উদাহরণ | ওয়েব ব্রাউজার, ইমেইল, ফাইল ট্রান্সফার |

---

### 🟣 Layer 6: Presentation Layer (প্রেজেন্টেশন)

```
ডেটা: "Hello"
      ↓
[এনক্রিপশন] → "X7#kL9"
[ফরম্যাট] → JPEG, PNG, UTF-8
[কম্প্রেশন] → ছোট সাইজ
```

| বৈশিষ্ট্য | বিবরণ |
|----------|--------|
| কাজ | ডেটা ফরম্যাট, এনক্রিপশন, কম্প্রেশন |
| প্রোটোকল | SSL/TLS, JPEG, GIF, MPEG |
| উদাহরণ | HTTPS এর এনক্রিপশন, ইমেজ ফরম্যাট |

---

### 🟢 Layer 5: Session Layer (সেশন)

```
[ক্লায়েন্ট]                    [সার্ভার]
     │                              │
     │←── সেশন স্থাপন ──────────────→│
     │←── ডেটা আদান-প্রদান ─────────→│
     │←── সেশন বন্ধ ────────────────→│
```

| বৈশিষ্ট্য | বিবরণ |
|----------|--------|
| কাজ | সেশন তৈরি, রক্ষণাবেক্ষণ, বন্ধ করা |
| প্রোটোকল | NetBIOS, RPC, PPTP |
| উদাহরণ | লগইন সেশন, ভিডিও কল সেশন |

---

### 🟡 Layer 4: Transport Layer (ট্রান্সপোর্ট)

```
[বড় ডেটা]
    ↓
┌────────┬────────┬────────┐
│Segment1│Segment2│Segment3│  ← ছোট করে ভাগ
└────────┴────────┴────────┘
    │         │        │
   Port     Port     Port    ← পোর্ট নম্বর যোগ
```

| বৈশিষ্ট্য | বিবরণ |
|----------|--------|
| কাজ | End-to-end ডেলিভারি, ফ্লো কন্ট্রোল |
| প্রোটোকল | TCP (নির্ভরযোগ্য), UDP (দ্রুত) |
| ডেটা ইউনিট | Segment |

#### TCP vs UDP

| বৈশিষ্ট্য | TCP | UDP |
|----------|-----|-----|
| নির্ভরযোগ্যতা | হ্যাঁ | না |
| স্পিড | ধীর | দ্রুত |
| হ্যান্ডশেক | আছে | নেই |
| ব্যবহার | ওয়েব, ইমেইল | ভিডিও, গেম |

#### TCP Three-Way Handshake
```
Client                    Server
  │                          │
  │──── SYN ────────────────→│
  │                          │
  │←─── SYN + ACK ───────────│
  │                          │
  │──── ACK ────────────────→│
  │                          │
  │←─── ডেটা আদান-প্রদান ────→│
```

---

### 🟠 Layer 3: Network Layer (নেটওয়ার্ক)

```
[সোর্স PC]                           [ডেস্টিনেশন PC]
192.168.1.10                         10.0.0.50
     │                                    │
     └──→ [Router A] ──→ [Router B] ──→──┘
              ↓              ↓
         রাউটিং টেবিল    রাউটিং টেবিল
         সেরা পথ নির্বাচন সেরা পথ নির্বাচন
```

| বৈশিষ্ট্য | বিবরণ |
|----------|--------|
| কাজ | লজিক্যাল অ্যাড্রেসিং, রাউটিং |
| প্রোটোকল | IP, ICMP, ARP, OSPF, BGP |
| ডেটা ইউনিট | Packet |
| ডিভাইস | Router |

#### IP Header (সরলীকৃত)
```
┌────────────────────────────────────┐
│ Source IP      │ Destination IP   │
│ 192.168.1.10   │ 10.0.0.50        │
├────────────────┼──────────────────┤
│    TTL         │    Protocol      │
└────────────────┴──────────────────┘
```

---

### 🔵 Layer 2: Data Link Layer (ডেটা লিংক)

```
┌─────────────────────────────────────────┐
│ Ethernet Frame                           │
├──────────┬──────────┬────────┬─────────┤
│Dest MAC  │Source MAC│  Data  │  FCS    │
│(6 bytes) │(6 bytes) │        │(4 bytes)│
└──────────┴──────────┴────────┴─────────┘
```

| বৈশিষ্ট্য | বিবরণ |
|----------|--------|
| কাজ | ফ্রেম তৈরি, MAC অ্যাড্রেসিং, এরর ডিটেকশন |
| প্রোটোকল | Ethernet, Wi-Fi (802.11), PPP |
| ডেটা ইউনিট | Frame |
| ডিভাইস | Switch, Bridge |

#### MAC Address উদাহরণ
```
00:1A:2B:3C:4D:5E
└──┬──┘ └──┬──┘
   │       │
   OUI    Device ID
(প্রস্তুতকারক)(ডিভাইস)
```

---

### ⚪ Layer 1: Physical Layer (ফিজিক্যাল)

```
ডেটা: 1 0 1 1 0 1 0 0
        ↓
┌───────────────────────┐
│   ↗   ↘   ↗ ↗   ↘ ↗   │  ← বৈদ্যুতিক সিগন্যাল
│  ▔▔▔ ▁▁▁ ▔▔▔▔ ▁▁▁ ▔▔  │
└───────────────────────┘
```

| বৈশিষ্ট্য | বিবরণ |
|----------|--------|
| কাজ | বিট ট্রান্সমিশন (0/1) |
| মিডিয়া | ক্যাবল, ফাইবার, রেডিও |
| ডেটা ইউনিট | Bits |
| ডিভাইস | Hub, Repeater, Cable, NIC |

---

## 📦 ডেটা Encapsulation (অত্যন্ত গুরুত্বপূর্ণ!)

যখন ডেটা পাঠানো হয়, প্রতিটি লেয়ার নিজের header যোগ করে:

```
Application Data: "Hello World"
        ↓
┌─────────────────────────────────────┐
│ L7-L5 │         Data                │
└─────────────────────────────────────┘
        ↓
┌──────┬──────────────────────────────┐
│TCP/UDP│         Data                │ → Segment
│Header │                             │
└──────┴──────────────────────────────┘
        ↓
┌──────┬──────┬───────────────────────┐
│  IP  │TCP/UDP│       Data           │ → Packet
│Header│Header │                      │
└──────┴──────┴───────────────────────┘
        ↓
┌──────┬──────┬──────┬────────┬──────┐
│ MAC  │  IP  │TCP/UDP│  Data  │ FCS │ → Frame
│Header│Header│Header │        │     │
└──────┴──────┴──────┴────────┴──────┘
        ↓
    0 1 0 1 1 0 1 0 1 0 1 1 0 ...    → Bits
```

### PDU (Protocol Data Unit) নাম

| Layer | PDU নাম |
|-------|---------|
| Layer 7-5 | Data |
| Layer 4 | Segment (TCP) / Datagram (UDP) |
| Layer 3 | Packet |
| Layer 2 | Frame |
| Layer 1 | Bits |

## 🧰 প্রতি লেয়ারে ডিভাইস

```
Layer 7: [Web Server] [Email Server] [Application]
            ↓

            ↓
Layer 3: [Router] [L3 Switch]
            ↓
Layer 2: [Switch] [Bridge] [Access Point]
            ↓
Layer 1: [Hub] [Repeater] [Cable] [NIC]
```

## 🩺 OSI দিয়ে ট্রাবলশুটিং

### সমস্যা: ইন্টারনেট কাজ করছে না

```
Step 1 (Layer 1): কেবল ঠিক আছে? লিংক লাইট জ্বলছে?
        ↓ হ্যাঁ
Step 2 (Layer 2): MAC পাচ্ছে? VLAN ঠিক আছে?
        ↓ হ্যাঁ
Step 3 (Layer 3): IP আছে? Gateway ping হয়?
        ↓ হ্যাঁ
Step 4 (Layer 4): Port open? Firewall block করছে?
        ↓ হ্যাঁ
Step 5 (Layer 7): DNS কাজ করছে? Application error?
```

### Quick Commands (Windows)

| Layer | Check করার উপায় |
|-------|------------------|
| L1 | Link light, `ipconfig` (Media disconnected?) |
| L2 | `arp -a` (MAC দেখা) |
| L3 | `ping 192.168.1.1` (Gateway) |
| L3 | `ping 8.8.8.8` (Internet) |
| L4 | `netstat -an` (Port check) |
| L7 | `nslookup google.com` (DNS) |

## 📊 সম্পূর্ণ রেফারেন্স টেবিল

| Layer | নাম | কাজ | প্রোটোকল | PDU | ডিভাইস |
|-------|-----|-----|----------|-----|--------|
| 7 | Application | ইউজার সার্ভিস | HTTP, DNS, FTP | Data | Server |
| 6 | Presentation | ফরম্যাট/এনক্রিপ্ট | SSL/TLS, JPEG | Data | - |
| 5 | Session | সেশন ম্যানেজ | NetBIOS, RPC | Data | - |
| 4 | Transport | End-to-end | TCP, UDP | Segment | Firewall |
| 3 | Network | রাউটিং | IP, ICMP | Packet | Router |
| 2 | Data Link | MAC, ফ্রেম | Ethernet, Wi-Fi | Frame | Switch |
| 1 | Physical | বিট | - | Bits | Hub, Cable |
