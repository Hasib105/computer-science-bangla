## 🔐 CIA Triad (সিকিউরিটির তিন স্তম্ভ)

```
                    ▲
                   /│\
                  / │ \
                 /  │  \
                /   │   \
               / Confidentiality
              /     │     \
             /      │      \
            /───────┼───────\
           /        │        \
          /         │         \
         /          │          \
        /           │           \
       /────────────┼────────────\
      │  Integrity  │  Availability │
      └─────────────┴───────────────┘
```

| নীতি | বাংলা | অর্থ |
|------|-------|------|
| **Confidentiality** | গোপনীয়তা | শুধু authorized ব্যক্তি দেখতে পারবে |
| **Integrity** | অখণ্ডতা | Data পরিবর্তন হয়নি ensure করা |
| **Availability** | প্রাপ্যতা | যখন দরকার তখন accessible থাকা |

### বাস্তব উদাহরণ

```

  ATM 24/7 কাজ করে
```

### ৩. Man-in-the-Middle (MitM)

```
Normal:
[Client] ──────────────────────→ [Server]
              Direct communication

MitM Attack:
[Client] ────→ [Attacker] ────→ [Server]
         ←────           ←────
         
Attacker reads/modifies all traffic!
```

## 🛡️ Defense in Depth (বহুস্তরীয় প্রতিরক্ষা)

একটা layer fail হলেও অন্যগুলো protect করবে।

```
┌─────────────────────────────────────────────────────┐
│               Physical Security                      │ ← Layer 1
│  ┌───────────────────────────────────────────────┐  │
│  │            Perimeter Security                 │  │ ← Layer 2
│  │  ┌─────────────────────────────────────────┐  │  │
│  │  │         Network Security                │  │  │ ← Layer 3
│  │  │  ┌───────────────────────────────────┐  │  │  │
│  │  │  │       Endpoint Security           │  │  │  │ ← Layer 4
│  │  │  │  ┌─────────────────────────────┐  │  │  │  │
│  │  │  │  │     Application Security    │  │  │  │  │ ← Layer 5
│  │  │  │  │  ┌───────────────────────┐  │  │  │  │  │
│  │  │  │  │  │    Data Security      │  │  │  │  │  │ ← Layer 6
│  │  │  │  │  │      🔒 DATA 🔒       │  │  │  │  │  │
│  │  │  │  │  └───────────────────────┘  │  │  │  │  │
│  │  │  │  └─────────────────────────────┘  │  │  │  │
│  │  │  └───────────────────────────────────┘  │  │  │
│  │  └─────────────────────────────────────────┘  │  │
│  └───────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

| Layer | Controls |
|-------|----------|
| Physical | Locks, CCTV, Access cards |
| Perimeter | Firewall, DMZ |
| Network | Segmentation, VPN |
| Endpoint | Antivirus, Patching |
| Application | Input validation, WAF |
| Data | Encryption, Backup |

## 🔒 Encryption (এনক্রিপশন)

### Symmetric vs Asymmetric

```
Symmetric (একই Key):
┌──────┐   Key=ABC   ┌──────┐   Key=ABC   ┌──────┐
│Sender│ ──────────→ │Cipher│ ──────────→ │Receiver│
│Plain │   Encrypt   │Text  │   Decrypt   │Plain   │
└──────┘             └──────┘             └──────┘
    Same key both sides (AES, DES)


Asymmetric (দুইটা Key):
┌──────┐  Public Key  ┌──────┐  Private Key ┌──────┐
│Sender│ ──────────→  │Cipher│ ──────────→  │Receiver│
│Plain │   Encrypt    │Text  │   Decrypt    │Plain   │
└──────┘              └──────┘              └──────┘
    Anyone's public      Only receiver's private
    key encrypts         key decrypts (RSA)
```

| Type | Speed | Key Distribution | Use Case |
|------|-------|------------------|----------|
| Symmetric | Fast | Difficult | Bulk data |
| Asymmetric | Slow | Easy | Key exchange, Digital signatures |

### Common Algorithms

```
Symmetric:          Asymmetric:
├── AES (best)      ├── RSA
├── 3DES            ├── ECC
└── Blowfish        └── Diffie-Hellman
```

## 📡 Wireless Security

### WiFi Security Evolution

```
WEP → WPA → WPA2 → WPA3
 ✗     △      ✓      ✓✓

WEP:  Cracked in minutes ❌
WPA:  TKIP, still weak
WPA2: AES, current standard ✓
WPA3: Latest, most secure ✓✓
```

### Best Practices

```
✓ Use WPA2/WPA3 with strong password
✓ Hide SSID (optional security)
✓ MAC filtering (limited use)
✓ Guest network separate করুন
✓ Regularly change password
✗ Open networks avoid করুন
```
