# SDN এবং NFV (Software Defined Networking & Network Function Virtualization)

## 📖 SDN কী?

**SDN (Software Defined Networking)** হলো এমন একটি approach যেখানে network এর **control plane** (brain) কে **data plane** (muscle) থেকে আলাদা করা হয় এবং centrally manage করা হয়।

```
Traditional Network:              SDN:

┌────────┐ ┌────────┐            ┌─────────────────────┐
│Router 1│ │Router 2│            │   SDN Controller    │
│ Brain  │ │ Brain  │            │   (Central Brain)   │
│   +    │ │   +    │            └──────────┬──────────┘
│ Muscle │ │ Muscle │                       │
└────────┘ └────────┘             ┌─────────┼─────────┐
                                  ↓         ↓         ↓
Each device                   ┌──────┐ ┌──────┐ ┌──────┐
thinks for itself             │Switch│ │Switch│ │Switch│
                              │Muscle│ │Muscle│ │Muscle│
                              └──────┘ └──────┘ └──────┘
                              Only forward, don't think!
```

## 🔌 OpenFlow Protocol

SDN Controller এবং switches এর মধ্যে communication।

```
OpenFlow Message Types:

Controller → Switch:
├── Modify-State: Flow table update করো
├── Read-State: Statistics দাও
├── Packet-Out: এই packet forward করো
└── Barrier: সব command execute হয়েছে confirm করো

Switch → Controller:
├── Packet-In: Unknown packet, কী করবো?
├── Flow-Removed: Flow timeout/delete হয়েছে
└── Port-Status: Port state change হয়েছে
```

## 📖 NFV কী?

**NFV (Network Function Virtualization)** হলো dedicated hardware devices (firewall, load balancer, router) এর বদলে **software** হিসেবে virtual machines এ run করা।

```
Traditional (Hardware-based):

┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐
│Firewall │ │   LB    │ │  WAN    │ │  IDS    │
│Hardware │ │Hardware │ │Optimizer│ │Hardware │
└─────────┘ └─────────┘ └─────────┘ └─────────┘
   $$$         $$$         $$$         $$$
Dedicated appliances - expensive!


NFV (Virtualized):

┌─────────────────────────────────────────────────────────┐
│                  Standard x86 Server                     │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐   │
│  │  vFW     │ │  vLB     │ │  vWAN    │ │  vIDS    │   │
│  │  VM      │ │  VM      │ │  Opt VM  │ │  VM      │   │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘   │
│                                                          │
│         Hypervisor (VMware, KVM, etc.)                  │
│                                                          │
│                Standard Server Hardware                  │
└─────────────────────────────────────────────────────────┘
                   One server, many functions!
```

### Key Terms

| Term | Full Form | Role |
|------|-----------|------|
| VNF | Virtual Network Function | Virtual firewall, LB, etc. |
| NFVI | NFV Infrastructure | Compute, storage, network resources |
| VIM | Virtualized Infrastructure Manager | OpenStack, VMware |
| VNFM | VNF Manager | VNF lifecycle manage করে |
| NFVO | NFV Orchestrator | Overall orchestration |

## 🆚 SDN vs NFV

| Aspect | SDN | NFV |
|--------|-----|-----|
| Focus | Network control | Network functions |
| Separates | Control from Data plane | Software from Hardware |
| Main benefit | Programmability | Flexibility |
| Key protocol | OpenFlow | ETSI MANO |
| Origin | Stanford/ONF | Telecom operators |
| Example | SDN Controller | Virtual Firewall |

**মনে রাখুন:**
- SDN = HOW traffic flows
- NFV = WHAT processes traffic
