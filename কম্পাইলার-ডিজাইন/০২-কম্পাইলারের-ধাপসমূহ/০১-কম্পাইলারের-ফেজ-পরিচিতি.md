# 🔄 কম্পাইলারের ফেজ পরিচিতি (Phases of Compiler)

কম্পাইলার সোর্স কোডকে মেশিন কোডে রূপান্তর করতে ৬টি প্রধান ধাপ অনুসরণ করে।

│       │                                                         │
│       ▼                                                         │
│  ┌─────────────────────┐                                        │
│  │ ১. Lexical Analysis │  টোকেনে ভাগ করা                        │
│  └──────────┬──────────┘                                        │
│             │                                                   │
│             ▼                                                   │
│  ┌─────────────────────┐                                        │
│  │ ২. Syntax Analysis  │  পার্স ট্রি তৈরি                        │
│  └──────────┬──────────┘                                        │
│             │                                                   │
│             ▼                                                   │
│  ┌─────────────────────┐                                        │
│  │ ৩. Semantic Analysis│  অর্থগত যাচাই                          │

│             ▼                                                   │

│             │                                                   │

## 📊 ধাপগুলোর সংক্ষিপ্ত বিবরণ

| ধাপ | নাম | ইনপুট | আউটপুট |
|-----|-----|-------|--------|
| ১ | Lexical Analysis | সোর্স কোড | Tokens |
| ২ | Syntax Analysis | Tokens | Parse Tree |
| ৩ | Semantic Analysis | Parse Tree | Annotated Tree |
| ৪ | Intermediate Code | Annotated Tree | IR Code |
| ৫ | Code Optimization | IR Code | Optimized IR |
| ৬ | Code Generation | Optimized IR | Machine Code |

## 🔍 উদাহরণ দিয়ে বোঝা

### সোর্স কোড:
```c
int result = a + b * 5;
```

### ধাপ ১: Lexical Analysis
```
Tokens:
├── <keyword, int>
├── <identifier, result>
├── <operator, =>
├── <identifier, a>
├── <operator, +>
├── <identifier, b>
├── <operator, *>
└── <number, 5>
```

### ধাপ ২: Syntax Analysis
```
         =
        / \
   result  +
          / \
         a   *
            / \
           b   5
```

### ধাপ ৩: Semantic Analysis
```
- result: int টাইপ
- a: int টাইপ (Symbol Table থেকে)
- b: int টাইপ
- 5: int literal
- int + int = int ✓
- int * int = int ✓
```

### ধাপ ৪: Intermediate Code
```
t1 = b * 5
t2 = a + t1
result = t2
```

### ধাপ ৫: Code Optimization
```
t1 = b * 5    // b শিফট করা যায় যদি 5 = 2^n-1 না হয়
t2 = a + t1
result = t2   // সরাসরি result-এ assign
```

### ধাপ ৬: Code Generation
```asm
MOV R1, b
MUL R1, 5
ADD R1, a
MOV result, R1
```

## 🏗️ Front End vs Back End

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │                    FRONT END                              │  │
│  │  (ভাষা নির্ভর, মেশিন স্বাধীন)                               │  │
│  │                                                           │  │
│  │  Lexical → Syntax → Semantic → Intermediate Code          │  │
│  └───────────────────────────────────────────────────────────┘  │
│                             │                                   │
│                             ▼                                   │
│                       Symbol Table                              │
│                             │                                   │
│                             ▼                                   │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │                    BACK END                               │  │
│  │  (মেশিন নির্ভর, ভাষা স্বাধীন)                               │  │
│  │                                                           │  │
│  │  Code Optimization → Code Generation                      │  │
│  └───────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 📚 দুটি সহায়ক উপাদান

### ১. Symbol Table (সিম্বল টেবিল)
```
┌────────────┬──────────┬────────┬─────────┐
│    নাম     │   টাইপ   │ স্কোপ  │ মেমরি   │
├────────────┼──────────┼────────┼─────────┤
│   result   │   int    │ main   │  1000   │
│     a      │   int    │ main   │  1004   │
│     b      │   int    │ main   │  1008   │
└────────────┴──────────┴────────┴─────────┘
```

### ২. Error Handler (এরর হ্যান্ডলার)
```
┌─────────────────────────────────────────┐
│  এরর প্রকার:                            │
│  - Lexical Error: অবৈধ টোকেন            │
│  - Syntax Error: ব্যাকরণ ভুল            │
│  - Semantic Error: টাইপ মিসম্যাচ        │
│  - Runtime Error: রানটাইম সমস্যা        │
└─────────────────────────────────────────┘
```

## 🔄 Analysis vs Synthesis

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│        ANALYSIS (বিশ্লেষণ)          SYNTHESIS (সংশ্লেষণ)         │
│                                                                 │
│     ┌─────────────────┐         ┌─────────────────┐            │
│     │ Lexical         │         │ Intermediate    │            │
│     │ Syntax          │         │ Optimization    │            │
│     │ Semantic        │         │ Code Generation │            │
│     └────────┬────────┘         └────────┬────────┘            │
│              │                           │                      │
│              ▼                           ▼                      │
│        সোর্স কোড বোঝা             টার্গেট কোড তৈরি               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 💡 মনে রাখার বিষয়

| ধাপ | মূল কাজ | ত্রুটি উদাহরণ |
|-----|--------|--------------|
| Lexical | টোকেন তৈরি | `@#$` অবৈধ ক্যারেক্টার |
| Syntax | গঠন যাচাই | `if (a+)` অসম্পূর্ণ |
| Semantic | অর্থ যাচাই | `int x = "hello"` টাইপ এরর |
| Intermediate | IR তৈরি | - |
| Optimization | উন্নতি | - |
| Code Gen | মেশিন কোড | - |
